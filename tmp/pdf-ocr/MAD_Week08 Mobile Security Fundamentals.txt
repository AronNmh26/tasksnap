Mission 2: Hardening Secrets — The .env Solution

If you found a hardcoded secret, the solution is to extract it from the code. We use environment
variables, typically managed in a .env file, to load secrets at runtime.

The Fix - A Two-Step Process

1. Create a .env file in your project's root directory. This file will store your secrets. Example .env
file content:
API_KEY="sk-aBcDeFg12345HiJkLmNoPgRs..."

2. Add .env to your .gitignore file. This is the most critical step. It prevents Git from ever
committing your secrets to the repository. If you forget this, your secrets will be exposed on
GitHub.
