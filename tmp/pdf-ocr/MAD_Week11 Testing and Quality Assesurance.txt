Part 2: Generating and Implementing a Mock
Objective: To test the logic of your “UserRepository” (e.g., JSON parsing) without making a
real network call.
Al Prompt
I need to test my UserRepository class. It depends on an ApiClient. Help me write a
test that Mocks the ApiClient.
1. Use [Mockito / Mocktail / Jest Mocks].
2. Stub the get() method to return a fake JSON response: {'name': 'Test User'}.
3. Verify that UserRepository parses this JSON correctly into a User object.
A Common Pitfall: Dependency Injection is Required!
Mocking only works if you can replace the dependency. If your code creates its
*ApiClient’ inside a method, you can't mock it.
Solution: Pass the client in via the constructor. This allows you to provide the real
client in the app and the mock client in the test.
